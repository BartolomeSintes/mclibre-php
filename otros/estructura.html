Contenido del curso

El curso se encuentra dividido en los siguientes apartados:

  1. Puesta en marcha: Instalación y configuración del servidor y del editor.
  2. PHP, sin formularios: Primeras páginas en PHP, Variables (enteros, decimales y cadenas), Funciones, Estructuras de control if ... elseif ... else ... y bucle for, Matrices.
  3. PHP, con formularios: Recogida, comprobación y utilización de datos proporcionados por el usuario en formularios.
  4. PHP, con sesiones: Cabeceras HTTP, sesiones.
  5. PHP, con bases de datos: Acceso a bases de datos desde PHP.

Un navegador web se puede considerar desde dos puntos de vista:

  • Los navegadores son por un lado visualizadores de documentos, las páginas web. Esas páginas web pueden contener texto, imágenes o vídeos y se conectan unas con otras mediante hiperenlaces.
  • Los navegadores son por otro lado entornos de ejecución de aplicaciones. Estas aplicaciones pueden trabajar con datos locales o remotos, aisladas o conectadas a otros navegadores, pero se ejecutan en el navegador.

En sus inicios, la web era únicamente una colección de documentos, pero cada vez más, la web está formada por una colección de aplicaciones.

Para que una aplicación se pueda ejecutar en un navegador, las aplicaciones deben estar escritas en el lenguaje de programación JavaScript. Ese lenguaje se creó inicialmente para que los navegadores pudieran realizar pequeñas manipulaciones en las páginas web, y por eso los programas JavaScript en principio deben incluirse dentro de una página web. Pero, con el tiempo, la mejora del lenguaje ha permitido crear aplicaciones tan complejas como cualquier aplicación de escritorio. En la actualidad, a menudo la página web que contiene el programa JavaScript apenas incluye contenido, ya que todo el contenido lo genera el programa JavaScript.

Cuando creamos un sitio web de páginas web (no una aplicación), lo normal es ampliarlo añadiendo cada vez más páginas web, a medida que añadimos contenido. Sin embargo cuando creamos una aplicación, la tendencia es a ir ampliando el programa JavaScript, pero manteniendo una única página web.

En este curso vamos a aprender el lenguaje de programación PHP, un lenguaje que ha estado ligado a la web desde prácticamente sus inicios. PHP es un lenguaje de servidor, es decir, los programas de PHP se ejecutan en el servidor, no en el navegador. Los programas PHP lo que hacen es generar páginas web, y estas páginas se envían al navegador.

En este curso no vamos a utilizar el lenguaje de programación JavaScript, por lo que las páginas web que crearemos con PHP serán páginas web "puras", sin JavaScript incoporado. Aún así, a medida que veamos características de PHP, iremos creando conjuntos de páginas que imitarán el comportamiento de las aplicaciones JavaScript.

Primera parte. PHP sin formularios

Las primeras lecciones del curso son una introducción a los elementos básicos del lenguaje PHP, que son los de cualquier lenguaje de programación.

Una particularidad que tenemos que tener siempre presente es que cuando estamos viendo en el navegador una página con la extensión .php nuestro navegador no está ejecutando un programa PHP. En realidad, lo que hace nuestro navegador es pedirle al servidor que ejecute el programa PHP y el servidor nos entrega el resultado del programa. El resultado del programa será casi siempre una página web html porque eso es lo que puede mostrarnos el navegador.

Otro detalle importante a tener en cuenta es que al programar y probar los programas estamos jugando dos papeles muy distintos. Al programar tenemos que pensar que estamos en el servidor, porque los programas se ejecutan en el servidor. Pero al probar los programas en el navegador tenemos que pensar que no estamos en el servidor sino en un cliente, porque el resultado de los programas se visualiza en el navegador. En nuestro caso, el servidor y el navegador están en el mismo ordenador, pero normalmente no es así: los programas PHP están normalemente en un único servidor, pero los navegadores pueden estar en cualquier ordenador de escritorio o dispositivo móvil del mundo.

Una limitación de los programas de esta primera parte es que los programas no pueden pedir datos a los usuarios. Más adelante en el curso, veremos cómo puede un programa PHP recoger información del usuario y personalizar la página web en consecuencia, o conseguir información de bases de datos para construir la página web. Para que los programas no muestren Pero en las páginas de ejercicios de la primera parte utilizaremos la función rand(), para generar valores aleatorios que utilizaremos para crear resultados distintos en cada ejecución de los programas. Una vez abierta la página en el navegador, al actualizar la página en el navegador, obtendremos una página ligeramente distinta, la señal inequívoca de que la página se está creando por programa.

En los ejercicios Primeras páginas veremos los programas más sencillos, en los que crearemos línea a línea la página web con la instrucción print.

Segunda parte. PHP con formularios

En esta segunda parte trabajaremos con formularios. Los formularios son los elementos de las páginas web que permiten al usuario introducir información. Al hacer clic en el botón enviar, el navegador solicita al servidor otra página web .php y acompaña la petición con la información introducida por el usuario. El servidor ejecuta el programa PHP

Restos

Si las páginas web de nuestros programas fueran siempre iguales, entonces no haría falta usar PHP, sería suficiente con crear una página web html (no un programa PHP).
