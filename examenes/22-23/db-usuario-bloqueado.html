Exámenes - Usuario bloqueado

  • Comentarios

En este ejercicio se debe ampliar la aplicación proporcionada en la plantilla de manera que al crear un usuario se pueda indicar si el usuario está o no bloqueado. Un usuario bloqueado no podrá acceder a la aplicación. La aplicación debe funcionar tanto en SQLite como en MySQL. Haga una captura de la pantalla en MySQL en la que se vea la base de datos de la aplicación y el contenido de la tabla de usuarios.

Usuario bloqueado - Comentarios

Campo "bloqueado" en la tabla Usuarios

  • La tabla de Usuarios incluirá un nuevo campo llamado "bloqueado" que podrá tener dos valores, uno que represente que el usuario está bloqueado y otro que no (pueden ser valores numéricos o cadenas, a elección del alumno, pero se aconseja utilizar valores numéricos).
  • Los usuarios iniciales (tanto el usuario root como los del fichero demo.php) no estarán bloqueados.

Creación de usuarios

  • Al crear un usuario, se deberá indicar si el usuario está inicialmente bloqueado. El valor por defecto del botón radio será No.
