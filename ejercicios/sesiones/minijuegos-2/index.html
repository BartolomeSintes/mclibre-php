Se pueden descargar unas posibles soluciones de estos ejercicios en la página de soluciones.

Nota:

  • Los datos introducidos por el usuario deben guardarse en variables de sesión, no en controles ocultos (hidden).
  • Los valores introducidos por el usuario se deben comprobar y, si los valores no son correctos, se debe indicar el motivo al usuario (básicamente, haberse dejado los campos en blanco, pero también podría comprobarse la longitud de la cadena enviada).

Sesiones (1) 11 - Tirar dados

Escriba un programa de dos páginas que muestre un valor numérico y permita subirlo o bajarlo mediante dos botones. Ese valor numérico es el número de dados que se muestran en la página.

  • La primera página contiene un formulario con tres botones de tipo submit con el mismo name.
  • La segunda página recibe el dato, modifica el número (el valor mínimo es 1) y redirige a la primera página.
  • El número se guarda en una variable de sesión. Si la variable no está definida, se le dará el valor 1.
  • Los dados que se muestran en la primera página corresponden a entidades de carácter: de &#9856; ⚀ a &#9861; ⚅
  • Los dados se generan con un bucle for

Sesiones (1) 12 - Carrera de coches (1)

Escriba un programa de dos páginas que muestre dos puntos sobre dos línea y permita moverlos mediante dos botones que tiran dos dados.

  • La primera página contiene un formulario con tres botones de tipo submit con el mismo name.
  • La segunda página recibe el dato, modifica el número (el valor mínimo es 1) y redirige a la primera página.
  • El número se guarda en una variable de sesión. Si la variable no está definida, se le dará el valor 1.
  • Los dados que se muestran en la primera página corresponden a entidades de carácter: de &#9856; ⚀ a &#9861; ⚅
  • Los dados se generan con un bucle for
