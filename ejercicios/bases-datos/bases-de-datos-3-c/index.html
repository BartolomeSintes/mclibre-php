Esta página de ejercicios está en elaboración.

En esta página de ejercicios construiremos un pequeño CMS que nos permita gestionar una web de noticias dedicadas a las aplicaciones web y al desarrollo web.

Para facilitar la realización de los ejercicios, se proporcionan unas plantillas de los ejercicios Bases de datos (3 C) (Bases de datos).

  • La plantilla del ejercicio 1 es la solución del ejercicio Bases de datos (3 B) 6.
  • Las plantillas del resto de ejercicios son la solución del ejercicio anterior.

Puede descargar unas posibles soluciones de estos ejercicios en la página de soluciones.

Bases de datos (3 C) - 1 - Aplicación básica

En este ejercicio crearemos la aplicación básica en la que las noticias se guardarán en un tabla. Cada noticia tendrá un título, un cuerpo, una categoría (por ejemplo, el nombre de la aplicación a la que está dedicada la noticia) y una fecha de creación.

  • Las consultas de creación de tablas en SQLite podrían ser las siguientes: 
    // SQLITE: Consulta de creación de tabla Usuarios
    $consulta = "CREATE TABLE $cfg[dbUsuariosTabla]  (
                 id INTEGER PRIMARY KEY,
                 usuario VARCHAR($cfg[dbUsuariosTamUsuario]) COLLATE NOCASE,
                 password VARCHAR($cfg[dbUsuariosTamPassword]) COLLATE NOCASE,
                 nivel INTEGER
                 )";

// SQLITE: Consulta de creación de tabla Noticias
    $consulta = "CREATE TABLE $cfg[dbNoticiasTabla]  (
                 id INTEGER PRIMARY KEY,
                 categoria VARCHAR($cfg[dbNoticiasTamCategoria]) COLLATE NOCASE,
                 titulo VARCHAR($cfg[dbNoticiasTamTitulo]) COLLATE NOCASE,
                 cuerpo VARCHAR($cfg[dbNoticiasTamCuerpo]) COLLATE NOCASE,
                 creado DATE
                 )";
  • El campo "cuerpo" contendrá el texto completo de la noticia. En el formulario de creación del registro de noticia es conveniente utilizar un control <textarea>. Para definir su tamaño, en el archivo de configuración comunes/config.php definiremos su tamaño horizontal columns y vertical rows: 
        print "            <td>Cuerpo:</td>\n";
    print "            <td><textarea name=\"cuerpo\" rows=\"$cfg[formNoticiasTamCuerpoY]\" cols=\"$cfg[formNoticiasTamCuerpoX]\"></textarea></td>\n";

    $cfg["formNoticiasTamCuerpoX"]   = 60;  // Tamaño X del área de texto Noticias > Cuerpo
$cfg["formNoticiasTamCuerpoY"]   = 5;   // Tamaño Y del área de texto Noticias > Cuerpo
  • La plantilla de este ejercicio es la solución del ejercicio Bases de datos (3 B) 6. Se puede reaprovechar la tabla personas [db/tabla-personas] convirtiéndola en la tabla noticias [db/tabla-noticias]. El campo de tipo fecha "nacido" de la tabla personas se puede reconvertir en el campo "creado" de la tabla noticias.

Bases de datos (3 C) - 2 - Tabla categorías

Modifique la aplicación de ejemplo para que las categorías se encuentren en una tabla "categorias" aparte.

Bases de datos (3 C) - 3 - Portada con noticias

Modifique la aplicación del ejercicio anterior para que en la portada se muestren las últimas noticias en orden cronológico inverso. El número máximo de noticias que se muestran en la portada se establecerá en una variable de configuración.

Bases de datos (3 C) - 4 - Portada con categorías

Modifique la aplicación del ejercicio anterior para que en la portada se pueda elegir ver las noticias de una categoría determinada. El número máximo de noticias de una categoría que se muestran en la portada será el mismo que cuando se ven todas las noticias.
