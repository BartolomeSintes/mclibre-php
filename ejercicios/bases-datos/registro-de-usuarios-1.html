Registro de usuarios

  • 1 - Aplicación básica
  • 2 - Ampliación

El objetivo de este proyecto es crear un sistema de registro de usuarios (con nombre y contraseña) que se utilizará en los proyectos siguientes.

Registro de usuarios 1 - Aplicación básica

Escriba un programa que permita crear una base de datos de nombres de usuario y contraseñas que permita también guardar registros.

Comentarios

  • El menú de la página index.php incluye las opciones:
    • Borrar todo: para borrar y crear la base de datos de usuarios
    • Registrar nuevo usuario: para introducir registros en la base de datos
  • El menú del resto de páginas incluye la opción:
    • Volver al inicio
  • Estas pueden ser las consultas de creación de tabla para este proyecto: 
    // Consultas de creación de tablas en MySQL
$consultaCreaTablaUsuarios = "CREATE TABLE $dbUsuarios (
    id INTEGER UNSIGNED NOT NULL AUTO_INCREMENT,
    usuario VARCHAR($tamUsuario),
    password VARCHAR($tamPassword),
    PRIMARY KEY(id) )";

// Consultas de creación de tablas en SQLite
$consultaCreaTablaUsuarios = "CREATE TABLE $dbUsuarios (
    id INTEGER PRIMARY KEY,
    usuario VARCHAR($tamUsuario),
    password VARCHAR($tamPassword)
    )";
  • En la página registrar1.php se solicita el nombre de usuario y su contraseña (por duplicado)
  • En la página registrar2.php se debe comprobar (y redirigir a registrar1.php con un mensaje de error) en las siguientes situaciones:
    • nombre de usuario en blanco
    • las contraseñas no coinciden
    • el nombre de usuario o la contraseña son demasiado largos
    • el nombre de usuario ya está registrado
    • se han creado ya el máximo de registros
  • En caso de que los datos sean correctos, registrar2.php debe guardar el nombre de usuario y la contraseña en un registro.

Registro de usuarios 2 - Ampliación

Amplie el ejercicio anterior:

  • Al crear un registro de usuario, no guarde la contraseña directamente, sino el md5 de la contraseña.

    Nota: Desde 2004, se desaconseja el uso de md5 para criptografía, aunque por sencillez se utilice en este ejercicio.

  • Al crear la tabla, guarde el registro de nombre root y contraseña root.
  • Añadir una opción de menú para identificarse como usuario. Al usuario identificado, se le ofrece únicamente la opción de menú:
    • Cerrar sesión
  • Un usuario identificado no puede ver ninguna página distinta a index.php y se le redirige a ella.

Comentarios

  • El md5 de la contraseña se puede calcular con la función md5().

    Nota: Desde 2004, se desaconseja el uso de md5 para criptografía, aunque por sencillez se utilice en este ejercicio.
